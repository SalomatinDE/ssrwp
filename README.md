# ssrwp
Техническое задание на разработку веб-приложения «Кулинарная платформа: Генератор меню»
# 1. Общие требования

Приложение должно представлять собой интерактивную платформу для поиска и хранения кулинарных рецептов с интуитивно понятным интерфейсом, адаптивным дизайном (desktop/tablet/mobile) и обеспечивать удобный доступ к функционалу на любых устройствах.

Стек технологий:

Frontend: React, React Router, Redux Toolkit / Context API.

Стили: CSS Modules / Styled Components / Tailwind CSS.

Бэкенд: Node.js/Express.

База данных: PostgreSQL / MongoDB.

Технические требования к окружению:
Поддержка минимально необходимых версий браузеров (Chrome >= 60, Firefox >= 55, Edge >= 79, Safari >= 12). Кроссплатформенность.

# 2. Функциональные требования

# 2.1. Каталог рецептов и Поиск

Главная страница (Лента):

Пользователь видит ленту рецептов с пагинацией или бесконечной прокруткой.

Каждая карточка рецепта содержит: фото блюда, название, краткое описание, время приготовления, рейтинг и имя автора.

Поиск и фильтрация:

Наличие поисковой строки для поиска по названию рецепта.

Расширенный поиск:

Поле ввода с автодополнением.

Возможность добавить несколько ингредиентов в список "Что у меня есть".

Результаты поиска должны ранжироваться по релевантности (чем больше ингредиентов совпало/меньше лишних, тем выше результат).

Фильтры (боковая панель или выпадающие списки): по категории (супы, десерты), по времени приготовления, по типу питания (веганское, безглютеновое).

Страница рецепта:

Детальное описание: фото, ингредиенты (со списком покупок), пошаговая инструкция.

Интеграция с кухонным таймером (см. пункт 2.3).

Блок с рейтингом и отзывами других пользователей.

Кнопка "Сохранить в коллекцию".

# 2.2. Пользовательские коллекции (Бэк — пользовательские коллекции)

Личный кабинет пользователя:

Раздел "Мои коллекции" (плейлисты). Возможность создавать, удалять и переименовывать папки (например, "Завтраки", "На ужин", "Любимое").

Возможность добавлять рецепты в коллекцию прямо со страницы рецепта или из каталога.

История:

Просмотр последних просмотренных рецептов.

# 2.3. Генератор меню и Таймер

Генератор меню:

Отдельная страница или виджет.

Пользователь выбирает ингредиенты из списка (или указывает "Что есть в холодильнике").

Указывает параметры: "Завтрак/Обед/Ужин", "Сложность", "Время на готовку".

Приложение генерирует список рецептов, которые можно приготовить из этих продуктов.

Кухонный таймер (Интерактив):

На странице рецепта рядом с шагами, где указано время (например, "Варить 15 минут"), должна быть интерактивная кнопка "Запустить таймер".

По нажатию открывается небольшой виджет таймера (всплывающее окно или панель), который запускает обратный отсчет.

Таймер должен продолжать работу даже при переключении между страницами приложения (глобальный стейт) и издавать звуковой сигнал по окончании.

Возможность запустить несколько таймеров одновременно (для разных блюд).

# 2.4. Рейтинги и взаимодействие

Система оценок:

Возможность поставить оценку рецепту по 5-звездочной шкале.

Возможность оставить текстовый отзыв/комментарий.

Отображение:

Средний рейтинг отображается в карточке рецепта и на его странице.

Сортировка в каталоге: "По популярности" (по убыванию рейтинга), "По новизне".

# 2.5. Персонализация и сохранение данных

Авторизация:

Регистрация и вход по email/паролю или через социальные сети (VK, Google) для сохранения коллекций и истории между сессиями.

Сохранение состояния (для гостя):

Если пользователь не авторизован, его избранные рецепты и запущенные таймеры должны сохраняться в localStorage браузера и восстанавливаться при повторном визите (с предложением зарегистрироваться, чтобы не потерять данные).

# 3. Дополнительные функциональные требования (необязательно, но желательно)

"Умный" поиск:

Поиск с учетом синонимов или морфологии русского языка.

Социальная интеграция:

Возможность поделиться рецептом в соцсетях (VK, Telegram, OK) с генерацией красивого превью (Open Graph теги).

Калькулятор порций:

На странице рецепта ползунок или кнопки "+/-" для изменения количества порций. Количество ингредиентов в списке должно пересчитываться автоматически.

Интерактивные подсказки:

Если пользователь ищет рецепт, а ингредиента нет в базе, предложить создать карточку нового рецепта (краудсорсинг) или отправить запрос администратору.
